# Recommended Minimum Documentation — MPS.BR Level E (Partially Defined)

The recommended minimum documentation for Level E. Adds Validation (VAL) and Verification (VER) on top of the Level F processes.

## Structure
- requirements/ — Functional, non-functional, use cases, traceability and change requests
- projects/ — Roadmap, schedule, risks, status and releases
- quality/ — Quality plan, test strategy, validation plan, verification plan and test report
- architecture/ — General and architectural decisions
- configuration/ — Configuration management plan, version control and baseline
- measurement/ — Metrics and indicators
