# Complete Documentation — MPS.BR Level E (Partially Defined)

The complete set of Level E documents. Adds Product Integration (ITP), plus Validation (VAL) and Verification (VER).

## Structure
- requirements/ — Functional, non-functional, use cases, user stories, business rules, traceability and change requests
- architecture/ — General, backend, mobile, web, data and architectural decisions
- projects/ — Roadmap, schedule, risks, status, releases and lessons learned
- quality/ — Quality plan, test strategy, verification plan, validation plan, review checklist, definition of done and test report
- integration/ — Integration plan, integration strategy and integration report
- configuration/ — CM plan, version control, baseline and configuration items
- measurement/ — Metrics, indicators, metrics report and measurement objectives
