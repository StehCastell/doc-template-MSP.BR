# Corrective Actions Log

## Objective

Record actions defined to eliminate the causes of problems, non-conformities or deviations identified during project development.

---

# Problem Description

Objectively describe the identified problem.

Example:

> Recurring defects related to data validation were found during functional testing.

---

# Identified Impact

Describe the impacts caused by the problem.

Example:

* Rework;
* Delivery delays;
* Reduced product quality.

---

# Cause Analysis

Record the identified root cause.

Example:

> Lack of standardized criteria for input validation during implementation.

---

# Defined Corrective Action

Describe the action that will be taken to fix the problem.

Example:

> Update the validation components and review similar features to ensure consistency.

# Verification Criterion
