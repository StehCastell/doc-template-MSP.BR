# 📅 Process Area — Projects

> This folder is **self-contained**: everything you need to understand the area is here.

## 📌 What this area is in MPS.BR
Planning, monitoring and control of the work. Maps to **Project Management (GPR)** in MR-MPS-SW.

## 🎯 What problem it solves
Solves **unpredictable deliveries**: without planning and tracking, deadlines slip and risks surprise.

## ✅ What belongs in this area
- Schedule and roadmap
- Status tracking
- Release planning
- Effort/schedule estimates

## 🚫 What does NOT belong in this area
- Technical scope (→ requirements)
- Release process (→ processes)
- Detailed risks (→ risk-management)

## 📈 How it evolves from level G to A

| Level MPS.BR | Evolution of the area |
|---|---|
| Level G — Partially Managed | Planned and monitored project. |
| Level F — Managed | Planning with basic measurement. |
| Level E — Partially Defined | Planning per standard process. |
| Level D — Largely Defined | Integrated project management. |
| Level C — Defined | Planning considers decisions and reuse. |
| Level B — Quantitatively Managed | Quantitative schedule forecasts. |
| Level A — Optimizing | Estimates calibrated by history. |
