# ✅ Process Area — Quality

> This folder is **self-contained**: everything you need to understand the area is here.

## 📌 What this area is in MPS.BR
Ensures product and process meet expectations. Maps to **Quality Assurance (GQA)**.

## 🎯 What problem it solves
Solves **late defects and an unfollowed process**.

## ✅ What belongs in this area
- Test strategy and plan
- Definition of done
- Test reports
- Process/product audits

## 🚫 What does NOT belong in this area
- What to test (→ requirements)
- Aggregated metrics (→ measurement)

## 📈 How it evolves from level G to A

| Level MPS.BR | Evolution of the area |
|---|---|
| Level G — Partially Managed | — (not applicable yet) |
| Level F — Managed | Planned and executed testing. |
| Level E — Partially Defined | Standardized test strategy. |
| Level D — Largely Defined | QA integrated into the process. |
| Level C — Defined | Covers decisions/reuse. |
| Level B — Quantitatively Managed | Quality with quantitative targets. |
| Level A — Optimizing | Defect prevention via causal analysis. |
