# 📅 Área de Processo — Projetos

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Planejamento, monitoramento e controle do trabalho. Corresponde à **Gerência de Projetos (GPR)** do MR-MPS-SW.

## 🎯 Qual problema ela resolve
Resolve **entregas imprevisíveis**: sem plano e acompanhamento, prazos estouram e riscos surpreendem.

## ✅ O que entra nesta área
- Cronograma e roadmap
- Acompanhamento de status
- Planejamento de lançamentos
- Estimativas de esforço/prazo

## 🚫 O que NÃO entra nesta área
- Escopo técnico (→ requisitos)
- Processo de release (→ processos)
- Riscos detalhados (→ riscos)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | Projeto planejado e acompanhado. |
| Nível F — Gerenciado | Planejamento com medição básica. |
| Nível E — Parcialmente Definido | Planejamento conforme processo padrão. |
| Nível D — Largamente Definido | Gestão integrada do projeto. |
| Nível C — Definido | Planejamento considera decisões e reutilização. |
| Nível B — Gerenciado Quantitativamente | Previsões quantitativas de prazo. |
| Nível A — Em Otimização | Estimativas calibradas por histórico. |
