# ⚙️ Área de Processo — Processos

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Como o trabalho é feito: desenvolvimento, mudanças, testes e release. Apoia a **Definição** e **Avaliação e Melhoria do Processo (AMP)**.

## 🎯 Qual problema ela resolve
Resolve o **'cada um do seu jeito'**.

## ✅ O que entra nesta área
- Processo de desenvolvimento
- Processo de requisitos
- Processo de testes
- Processo de mudanças/release

## 🚫 O que NÃO entra nesta área
- Padrões e templates (→ organizacao)
- Métricas (→ medicao)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | — (ainda não aplicável) |
| Nível F — Gerenciado | — (ainda não aplicável) |
| Nível E — Parcialmente Definido | — (ainda não aplicável) |
| Nível D — Largamente Definido | Processos definidos e padronizados. |
| Nível C — Definido | Processos cobrem decisões/reutilização. |
| Nível B — Gerenciado Quantitativamente | Processos sob controle estatístico. |
| Nível A — Em Otimização | Processos melhorados e inovados. |
