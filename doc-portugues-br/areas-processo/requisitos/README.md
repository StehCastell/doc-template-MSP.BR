# 📋 Área de Processo — Requisitos

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Trata da captura, especificação e gerência de requisitos. No MR-MPS-SW corresponde ao processo **Gerência de Requisitos (GRE)**: manter os requisitos acordados e rastreáveis.

## 🎯 Qual problema ela resolve
Resolve o **escopo ambíguo e mutável**, causa raiz da maioria do retrabalho.

## ✅ O que entra nesta área
- Requisitos funcionais e não funcionais
- Matriz de rastreabilidade
- Solicitações de mudança de escopo
- Regras de negócio e critérios de aceitação

## 🚫 O que NÃO entra nesta área
- Como construir (→ arquitetura)
- Cronograma (→ projetos)
- Casos de teste (→ qualidade)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | Requisitos documentados e gerenciados. |
| Nível F — Gerenciado | Requisitos sob controle de mudança formal. |
| Nível E — Parcialmente Definido | Processo de requisitos padronizado. |
| Nível D — Largamente Definido | Integrado ao processo organizacional. |
| Nível C — Definido | Requisitos ligados a decisões e reutilização. |
| Nível B — Gerenciado Quantitativamente | Volatilidade medida quantitativamente. |
| Nível A — Em Otimização | Análise causal reduz retrabalho de requisitos. |
