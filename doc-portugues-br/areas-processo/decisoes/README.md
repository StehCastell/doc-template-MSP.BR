# 🧭 Área de Processo — Decisões

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Análise e registro de decisões com critérios. Corresponde à **Gerência de Decisões (GDE)**.

## 🎯 Qual problema ela resolve
Resolve **decisões importantes tomadas sem critério nem registro**.

## ✅ O que entra nesta área
- Critérios de decisão
- Análise de alternativas
- Registro de decisões

## 🚫 O que NÃO entra nesta área
- Decisões de arquitetura de produto (→ arquitetura)
- Riscos (→ riscos)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | — (ainda não aplicável) |
| Nível F — Gerenciado | — (ainda não aplicável) |
| Nível E — Parcialmente Definido | — (ainda não aplicável) |
| Nível D — Largamente Definido | — (ainda não aplicável) |
| Nível C — Definido | Decisões analisadas com critérios formais. |
| Nível B — Gerenciado Quantitativamente | Decisões apoiadas por dados. |
| Nível A — Em Otimização | Processo decisório otimizado. |

## 🗂️ Arquivos desta área (evolução por nível)
- [`nivel-C.md`](nivel-C.md) — capacidade: Decisões analisadas com critérios formais.
- [`nivel-B.md`](nivel-B.md) — capacidade: Decisões apoiadas por dados.
- [`nivel-A.md`](nivel-A.md) — capacidade: Processo decisório otimizado.

## 💡 Exemplos de uso
- Comparar alternativas com critérios ponderados.
- Registrar a decisão e sua justificativa.

## 🔗 Relação com outras áreas
- **riscos** — decisões consideram riscos.
- **arquitetura** — decisões técnicas estruturais.
- **organizacao** — decisões viram ativo.
