# 🏗️ Área de Processo — Arquitetura

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Estrutura técnica da solução: componentes, camadas e decisões. Apoia o **Projeto e Construção do Produto**.

## 🎯 Qual problema ela resolve
Resolve **decisões técnicas implícitas e inconsistentes**.

## ✅ O que entra nesta área
- Visão geral da arquitetura
- Arquitetura por camada/plataforma
- Decisões arquiteturais (ADR)
- Restrições técnicas

## 🚫 O que NÃO entra nesta área
- O que o sistema faz (→ requisitos)
- Planejamento (→ projetos)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | — (ainda não aplicável) |
| Nível F — Gerenciado | Arquitetura documentada (opcional). |
| Nível E — Parcialmente Definido | Arquitetura do projeto definida. |
| Nível D — Largamente Definido | Padrões arquiteturais organizacionais. |
| Nível C — Definido | Arquitetura ligada a decisões/reutilização. |
| Nível B — Gerenciado Quantitativamente | Atributos arquiteturais medidos. |
| Nível A — Em Otimização | Arquitetura evolui por inovação. |
