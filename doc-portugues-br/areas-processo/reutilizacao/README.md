# ♻️ Área de Processo — Reutilização

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Gestão de ativos reutilizáveis (componentes, bibliotecas, padrões). Corresponde à **Gerência de Reutilização (GRU)**.

## 🎯 Qual problema ela resolve
Resolve o **retrabalho de reconstruir o que já existe**.

## ✅ O que entra nesta área
- Critérios de reutilização
- Catálogo/biblioteca de componentes
- Ativos reutilizáveis

## 🚫 O que NÃO entra nesta área
- Padrões organizacionais gerais (→ organizacao)
- Arquitetura (→ arquitetura)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | — (ainda não aplicável) |
| Nível F — Gerenciado | — (ainda não aplicável) |
| Nível E — Parcialmente Definido | — (ainda não aplicável) |
| Nível D — Largamente Definido | — (ainda não aplicável) |
| Nível C — Definido | Reutilização gerenciada com critérios. |
| Nível B — Gerenciado Quantitativamente | Reutilização medida (taxa de reuso). |
| Nível A — Em Otimização | Reutilização otimizada continuamente. |

## 🗂️ Arquivos desta área (evolução por nível)
- [`nivel-C.md`](nivel-C.md) — capacidade: Reutilização gerenciada com critérios.
- [`nivel-B.md`](nivel-B.md) — capacidade: Reutilização medida (taxa de reuso).

## 💡 Exemplos de uso
- Catalogar componentes reutilizáveis.
- Definir critérios para reaproveitar um ativo.

## 🔗 Relação com outras áreas
- **organizacao** — ativos são organizacionais.
- **arquitetura** — componentes reutilizáveis.
- **processos** — reutilização no fluxo.
