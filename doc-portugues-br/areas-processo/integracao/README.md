# 🔗 Área de Processo — Integração

> Esta pasta é **autossuficiente**: tudo o que você precisa para entender a área está aqui.

## 📌 O que é esta área no MPS.BR
Estratégia e processo de integrar componentes e sistemas. Apoia a **Integração do Produto**.

## 🎯 Qual problema ela resolve
Resolve **falhas de integração descobertas tarde**.

## ✅ O que entra nesta área
- Estratégia de integração
- Plano de integração
- Processo de integração
- Relatórios de integração

## 🚫 O que NÃO entra nesta área
- Arquitetura dos componentes (→ arquitetura)
- Testes funcionais (→ qualidade)

## 📈 Como evolui do nível G ao A

| Nível MPS.BR | Evolução da área |
|---|---|
| Nível G — Parcialmente Gerenciado | — (ainda não aplicável) |
| Nível F — Gerenciado | — (ainda não aplicável) |
| Nível E — Parcialmente Definido | Estratégia de integração definida. |
| Nível D — Largamente Definido | Processo de integração padronizado. |
| Nível C — Definido | Integração considera reutilização. |
| Nível B — Gerenciado Quantitativamente | Integração medida quantitativamente. |
| Nível A — Em Otimização | Integração otimizada e automatizada. |
