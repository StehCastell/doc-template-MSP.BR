# Lições Aprendidas

## Objetivo

Este documento tem como objetivo registrar conhecimentos adquiridos durante a execução do projeto, permitindo a identificação de boas práticas, oportunidades de melhoria e ações preventivas para projetos futuros.

As lições aprendidas contribuem para:

- Melhoria contínua dos processos;
- Redução de erros recorrentes;
- Compartilhamento de conhecimento;
- Evolução das práticas de desenvolvimento;
- Apoio à tomada de decisões futuras.

---

# Quando Registrar uma Lição Aprendida

Uma lição aprendida pode ser registrada quando ocorrer:

- Conclusão de uma release;
- Encerramento de uma fase do projeto;
- Resolução de um problema relevante;
- Identificação de uma boa prática;
- Identificação de um risco materializado;
- Ocorrência de falhas ou retrabalho significativo;
- Implementação de melhorias de processo.

---

# Categorias

As lições aprendidas podem ser classificadas nas seguintes categorias:

| Categoria | Descrição |
|------------|------------|
| Requisitos | Levantamento, análise e gestão de requisitos |
| Planejamento | Cronograma, estimativas e acompanhamento |
| Desenvolvimento | Implementação e arquitetura |
| Qualidade | Testes, revisão e validação |
| Processos | Fluxos de trabalho e governança |
| Comunicação | Interação entre envolvidos |
| Infraestrutura | Ambientes e ferramentas |
| Riscos | Gestão e mitigação de riscos |
| Implantação | Publicação e entrega do produto |

# Boas Práticas

- Registrar a lição o mais próximo possível do evento.
- Descrever fatos de forma objetiva.
- Evitar apontamento de responsabilidades pessoais.
- Focar em processos, práticas e resultados.
- Registrar tanto sucessos quanto falhas.
- Revisar periodicamente as lições registradas.
