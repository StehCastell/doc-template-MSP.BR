# Registro de Ações Corretivas

## Objetivo

Registrar ações definidas para eliminar as causas de problemas, não conformidades ou desvios identificados durante o desenvolvimento do projeto.

---

# Informações Gerais

| Campo            | Informação                        |
| ---------------- | --------------------------------- |
| Identificação    | AC-001                            |
| Data de Registro |                                   |
| Responsável      |                                   |
| Status           | Aberta / Em andamento / Concluída |

---

# Descrição do Problema

Descrever objetivamente o problema identificado.

Exemplo:

> Foram encontrados defeitos recorrentes relacionados à validação de dados durante os testes funcionais.

---

# Impacto Identificado

Descrever os impactos causados pelo problema.

Exemplo:

* Retrabalho;
* Atrasos nas entregas;
* Redução da qualidade do produto.

---

# Análise da Causa

Registrar a causa raiz identificada.

Exemplo:

> Ausência de critérios padronizados para validação de entradas durante a implementação.

---

# Ação Corretiva Definida

Descrever a ação que será executada para corrigir o problema.

Exemplo:

> Atualizar os componentes de validação e revisar funcionalidades semelhantes para garantir consistência.

# Critério de Verificação

Descrever como será verificado se a ação foi eficaz.

Exemplo:

> Não ocorrência de novos defeitos do mesmo tipo durante as próximas execuções de testes.

---

# Evidências

Registrar evidências da implementação da ação corretiva.

Exemplo:

* Commits realizados;
* Relatórios de testes;
* Atualizações na documentação.
